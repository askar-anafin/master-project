Comparative Analysis of Deep Learning Architectures for Multi-Label ECG Classification

1. Methodology

A. Hardware & Environment

All experiments were conducted on a high-performance workstation optimized for deep learning.

  • GPU: NVIDIA GeForce RTX 3060 (12GB VRAM) - Utilized for mixed-precision (FP16) training.
  • CPU: Multi-core Processor (e.g., Intel Core i7-12700K or equivalent) handling data loading and augmentation.
  • RAM: 32 GB DDR4 System Memory.
  • Software Stack:
  • OS: Windows 11 Pro
  • Framework: PyTorch 2.5 (CUDA 12.4 backend)
  • Libraries: NumPy, Pandas, Scikit-Learn, WFDB (PhysioNet).

B. Dataset Description (PTB-XL)

The PTB-XL ECG Database is used, comprising 21,837 clinical 12-lead ECG records from 18,885 patients.

  • Signal Properties: 10 seconds duration, sampled at 500 Hz (5000 time steps).
  • Lead Configuration: Standard 12-lead system (I, II, III, aVR, aVL, aVF, V1-V6).
  • Labels: 71 different ECG statements, mapped to 5 diagnostic superclasses:
    1. NORM: Normal Sinus Rhythm (9528 records)
    2. MI: Myocardial Infarction (5486 records)
    3. STTC: ST/T Change (5250 records)
    4. CD: Conduction Disturbance (4907 records)
    5. HYP: Hypertrophy (2655 records)
  • Class Balance: The dataset is imbalanced, necessitating robust evaluation metrics like F1-Score and AUPRC.

C. Data Preprocessing

To ensure signal quality and model stability, the following preprocessing pipeline was applied:

  1. Bandpass Filtering: 0.5 - 50 Hz (Butterworth filter) to remove baseline wander and powerline noise.
  2. Z-Score Normalization: Applied per lead ($\mu=0, \sigma=1$) to standardize amplitude.
  3. Splitting: Stratified Split ensuring patient separation:
    • Training: 80% (~17,400 records)
    • Validation: 10% (~2,100 records)
    • Testing: 10% (~2,100 records)

D. Model Architectures

1. CNN: ResNet-18 (1D Adapted)

A deep convolutional network adapted for 1D time-series data.

  • Input: $(Batch, 12, 5000)$
  • Initial Block: Conv1d ($k=15, s=2$) $\rightarrow$ BN $\rightarrow$ ReLU $\rightarrow$ MaxPool ($k=3, s=2$). Output channels: 64.
  • Residual Stages:
  • Stage 1: 2 blocks (64 filters, stride 1)
  • Stage 2: 2 blocks (128 filters, stride 2)
  • Stage 3: 2 blocks (256 filters, stride 2)
  • Stage 4: 2 blocks (512 filters, stride 2)
  • Aggregation: Global Average Pooling $\rightarrow$ Flatten.
  • Classifier: Linear Layer ($512 \rightarrow 5$ classes).
  • Total Parameters: ~3.8 Million.

2. GNN: Spatio-Temporal Graph (ST-ReGE)

A hybrid architecture combining temporal feature extraction with graph-based spatial reasoning.

  • Nodes: 12 (One per ECG lead).
  • Temporal Backbone: Shared ResNet-like 1D CNN applied to each node independently. Extracts a 256-dim feature vector per lead.
  • Graph Construction: A Learnable Adjacency Matrix ($12 \times 12$) initialized randomly and optimized via gradient descent to learn inter-lead correlations.
  • Graph Convolution: Two GCN layers (Hidden Dim = 256) with residual connections ($X' = \sigma(AXW) + X$).
  • Readout: Mean pooling over the 12 nodes $\rightarrow$ Linear Classifier.
  • Why it works: Captures "Infarct Localization" (e.g., Lead V1-V4 correlation for Anterior MI).

3. ViT: 1D Vision Transformer

A pure attention-based architecture treating the signal as a sequence of tokens.

  • Patch Embedding: Non-overlapping convolution ($k=50, s=50$). Splits 5000 time steps into 100 patches.
  • Dimensionality: $D_{model} = 128$.
  • Encoder:
  • Layers: 4
  • Heads: 4 (Multi-head Self-Attention)
  • Feed-Forward Network: Dim 256
  • Positional Embedding: Learnable parameters added to patch tokens.
  • Classifier: CLS token $\rightarrow$ LayerNorm $\rightarrow$ Linear Layer.

4. Spatio-Temporal Hybrid (ResNet-Transformer)

A hybrid model bridging the gap between convolutional inductive biases and sequence-level self-attention.

  • CNN Front-End: 3 stages of 1D ResNet (up to 256 filters, stride 2 downsampling) to compress sequence length from 5000 to 313.
  • Lead Attention: Squeeze-and-Excitation channel attention block to model lead correlations.
  • Transformer Back-End: 3-layer Transformer Encoder (d_model=256, 8 heads) to model global context.
  • Classifier: CLS token output $\rightarrow$ LayerNorm $\rightarrow$ 2-layer MLP classifier.

E. Evaluation Metrics

To rigorously assess model performance on the imbalanced multi-label PTB-XL dataset, we employed the following metrics:

  1. Macro F1-Score: The harmonic mean of Precision and Recall, calculated per class and then averaged.
  2. Macro AUROC (Area Under Receiver Operating Characteristic): Measures the model's ability to discriminate between positive and negative classes across all decision thresholds.
  3. Macro AUPRC (Area Under Precision-Recall Curve): Summarizes the trade-off between precision and recall for different thresholds.
  4. Exact Match Accuracy: The percentage of samples where the predicted set of labels exactly matches the true set of labels.
  5. PhysioNet Challenge Score: A specialized clinical utility metric. It penalizes false negatives (missed diagnoses) significantly more than false alarms (false positives).

2. Experimental Results

Training was performed for 100 epochs with early stopping (patience=10) on all models.

The Hybrid model represents a robust bridge, mitigating the training instability and data hunger of the pure ViT while keeping temporal relationship modeling intact.

3. Discussion

The results highlights a clear hierarchy in model suitability for ECG analysis:

  1. The Dominance of Convolutions: The CNN (ResNet-18) consistently performed best. ECG interpretation relies heavily on detecting specific local waveforms (P-waves, QRS complexes, T-waves) and their morphologies. Convolutional filters are mathematically equivalent to the cross-correlation operations used in traditional signal processing, making them the perfect "inductive bias" for this task.

  2. The Hybrid Bridge: The Spatio-Temporal Hybrid (ResNet-Transformer) shows that combining convolutions with transformers successfully addresses the data-hunger of attention networks. By using ResNet stages to extract local features and downsample the temporal length, the Transformer backend is fed stable, localized tokens, which prevented training collapse and achieved near-top-tier performance (F1: 0.7063, PhysioNet: 0.6998). In particular, it excelled at diagnosing **ST/T Changes (STTC)** where both shape and sequence behavior matter.

  3. The Promise of Graphs: The GNN (ST-ReGE) showed that explicitly modeling the geometry of the 12-lead system adds value, achieving high performance on conduction disturbances. However, its overall capability to learn purely morphological features is slightly lower than the CNN backbone.

  4. The Limitations of Transformers: The ViT underperformed because it lacked translation invariance and locality priors. Without large-scale pre-training, it struggled on sequential signals.

C. Threshold Optimization & Ensembling (Post-Hoc Enhancements)

To push diagnostic accuracy higher and address data imbalance without retraining, we evaluated two post-hoc strategies: class-specific validation threshold optimization and soft voting ensembling.

  1. Threshold Optimization: We optimized decision thresholds on the validation set using grid search over the range $[0.01, 0.99]$ with a step of $0.01$ to maximize the F1-score for each class independently (rather than using the standard static $0.5$ threshold).
  2. Soft Voting Ensemble: We averaged the predicted probabilities of the top three architectures (CNN + GNN + Hybrid).

Key Findings:

  • Ensemble Boost: The Soft Voting Ensemble achieved a Macro F1-score of 0.7398 (a 1.25% absolute improvement over the CNN baseline) and an Exact Match Accuracy of 64.50% (a 1.52% absolute improvement over the GNN baseline).
  • Ensemble Calibration: When combining ensembling with validation threshold optimization, the Ensemble reached a Macro F1 of 0.7439 (a total improvement of +1.66% over individual baselines) and a Macro AUROC of 0.9323.
  • Threshold Tuning: Optimizing thresholds boosted individual model F1-scores, helping the Spatio-Temporal Hybrid rise from 0.7063 to 0.7181 (+1.18%) and the ViT rise from 0.6776 to 0.6872 (+0.96%). Note that threshold optimization slightly decreases Exact Match Accuracy for some models because it changes the threshold from 0.5 (which is conservative and favors NORM) to class-specific values, leading to more active predictions and helping Macro F1 (macro average treats rare classes equally, so it benefits from predicting them, while Exact Match is extremely strict and penalizes any false positive).

4. Conclusion & Future Directions

This study provides a comprehensive evaluation of modern deep learning architectures for automated ECG diagnosis.

Final Verdict: The ResNet-18 (CNN) is the optimal model for immediate clinical deployment due to its high diagnostic accuracy (AUROC 0.9199). However, the Spatio-Temporal Hybrid represents a powerful new direction, demonstrating superior diagnostic accuracy on ST/T changes and high overall robustness.

Clinical Relevance: With a PhysioNet Score of 0.73, the CNN model provides a valuable "second opinion" for triage, safely filtering out Normal cases so experts can focus on pathologies.

Future Work:

  • Ensembling: A Voting Ensemble combining the CNN (sensitivity/morphology), GNN (spatial lead structures), and Hybrid (rhythm context) models would capture different feature dimensions, likely surpassing SOTA levels.
  • Pre-training: The ViT's and Hybrid's performance could be improved by self-supervised pre-training (e.g., Masked Auto-Encoders) on massive unlabeled datasets before fine-tuning.
  • Explainability: Integrate 1D Grad-CAM into the Hybrid model to verify which regions of the signal influence self-attention.
